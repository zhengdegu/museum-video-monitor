## Phase 4: 视频智能分析核心 ⭐

### Task 4.1: YOLO 人物检测 (US-4)
- [ ] 后端：yolo_detector.py — YOLO11 推理封装
- [ ] 实现跳帧粗扫（class=0 人物检测）
- [ ] 实现扩帧精扫（前后±5秒补漏）
- [ ] 实现短窗口合并（gap<3秒合并）
- [ ] 人物数量偏向值处理（80%置信度取最多人数）
- [ ] 输出 museum_person_segment 记录

### Task 4.2: 姿态与行为检测 (US-4)
- [ ] 后端：pose_tracker.py — YOLO-Pose + ByteTracker
- [ ] 奔跑检测：连续帧判定（避免转身误判）
- [ ] 运动轨迹分析
- [ ] 输出姿态结果集

### Task 4.3: 片段切分与抽帧 (US-4)
- [ ] 人物大片段按 60s 切分
- [ ] 每片段 1秒1帧 抽取截图
- [ ] 截图本地存储 + 路径入库
- [ ] 输出 museum_source_video_segment 记录

### Task 4.4: 大模型多模态分析 (US-4)
- [ ] 后端：llm_analyzer.py — vLLM 调用封装
- [ ] 每个 60s 片段截图 + YOLO/Pose/Tracker 结果送 Qwen3.5
- [ ] 增量合并结论（与前序结论压缩合并）
- [ ] Prompt 设计：着装识别、行为描述、人物交互

### Task 4.5: 裁判模型与规则匹配 (US-4, US-5)
- [ ] 后端：rule_engine.py — 规则匹配引擎
- [ ] 裁判模型 Prompt：结合合并结论 + 规则列表输出最终判定
- [ ] 生成 museum_event + museum_rule_hit 记录
- [ ] 生成 museum_event_aggregate 聚合记录

### Task 4.6: 分析调度器 (US-4)
- [ ] 后端：video_analyzer.py — 分析任务调度
- [ ] 视频状态流转管理（待分析→分析中→已完成/异常）
- [ ] 后台线程池消费分析任务
