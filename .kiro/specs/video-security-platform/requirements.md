# 博物馆视频智能监控分析平台 — 需求规格

## 项目概述

基于博物馆藏品安防场景，通过 YOLO11 + Qwen3.5 多模态大模型实现视频自动分析，管理端通过自然语言对话检索所有已分析视频的详细信息。全系统私有化部署，不联网不上云。

## 用户故事

### US-1: 库房管理
作为博物馆管理员，我需要管理多个库房信息，以便按库房维度组织摄像头和视频分析。

验收标准：
- [ ] 库房 CRUD（名称、编号、位置、描述）
- [ ] 库房详情页展示关联摄像头列表和分析状态概览
- [ ] 库房列表支持搜索和分页

### US-2: 摄像头管理
作为博物馆管理员，我需要管理每个库房下的摄像头设备，配置 RTSP 拉流地址和视频保存策略。

验收标准：
- [ ] 摄像头 CRUD（名称、RTSP 地址、所属库房、状态）
- [ ] 支持配置视频保存时长（默认 3 小时一段）
- [ ] 设备关联后支持自动全流程拉取
- [ ] 摄像头状态实时显示（在线/离线/拉流中）

### US-3: 视频管理
作为博物馆管理员，我需要管理原始视频，支持自动拉取和手动上传。

验收标准：
- [ ] 原始视频列表（来源、时长、大小、状态）
- [ ] 支持手动上传（大文件切块上传 + 分段上传）
- [ ] 自动拉取的视频默认存本地，分析完成后异步推送线上存储
- [ ] 视频状态流转：待分析 → 分析中 → 已完成 → 异常

### US-4: 视频智能分析
作为系统，我需要自动分析视频内容，检测人物、行为、着装，并生成结构化事件。

验收标准：
- [ ] YOLO11 跳帧粗扫检测人物（class=0）
- [ ] 人物区间前后扩帧精扫（±数秒补漏）
- [ ] 短窗口合并解决遮挡/光线导致的追踪断裂
- [ ] 人物大片段按 60s 切分，1 秒 1 帧抽图
- [ ] 每个切片送 Qwen3.5 多模态分析
- [ ] 增量合并结论（上下文压缩）
- [ ] 裁判模型 + 自定义规则输出最终事件结论
- [ ] 人物数量取偏向值（80% 置信度，取同画面最多人数）
- [ ] 奔跑检测用连续帧判定（避免转身误判）
- [ ] 姿态检测：YOLO-Pose + ByteTracker

### US-5: 规则引擎
作为博物馆管理员，我需要自定义安防规则，系统自动判定视频是否违规。

验收标准：
- [ ] 规则 CRUD（人数限制、着装要求、行为限制等）
- [ ] 预置 6 条核心规则：
  1. 进出库房人数 ≥ 2
  2. 统一工作服，不允许携带背包
  3. 手持文物双手持有，另一人在旁监督
  4. 禁止奔跑、跳跃、躲藏视角
  5. 只保留有人画面
  6. 自然语言检索事件
- [ ] 规则启用/禁用
- [ ] 规则命中统计

### US-6: 事件中心
作为博物馆管理员，我需要查看所有分析产生的事件，包括正常和违规事件。

验收标准：
- [ ] 事件列表（库房 + 摄像头 + 时间 + 事件详情）
- [ ] 事件聚合视图：按库房会话级聚合
- [ ] 规则命中快照：触发规则、时间、证据截图
- [ ] 事件时间线可视化
- [ ] 关联视频片段快速定位播放

### US-7: NLP 智能交互
作为博物馆管理员，我需要通过自然语言对话查询历史事件。

验收标准：
- [ ] ChatBot 对话界面
- [ ] 支持自然语言提问（如"昨天下午3号库房有没有人进出？"）
- [ ] 基于 RAG 从 Milvus 向量库检索
- [ ] 返回结果包含时间、库房、摄像头、事件详情、关联视频
- [ ] 支持多轮对话追问

### US-8: 藏品资料管理
作为博物馆管理员，我需要管理藏品信息、盘点和进出库记录。

验收标准：
- [ ] 藏品信息 CRUD（名称、编号、图片、描述、存放库房）
- [ ] 藏品盘点管理
- [ ] 进出库记录详情
- [ ] 藏品历史对比
- [ ] 盘点报表导出

### US-9: 系统管理
作为系统管理员，我需要管理用户、角色和权限。

验收标准：
- [ ] 用户管理（CRUD + 状态管理）
- [ ] 角色管理（RBAC）
- [ ] 动态路由菜单权限
- [ ] 操作日志审计
