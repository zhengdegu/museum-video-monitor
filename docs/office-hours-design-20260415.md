## Problem Statement

省级博物馆依赖传统监控系统（30路摄像头、2人值守），无法实时发现藏品被盗、损坏、游客受伤等安全事件。"出了事才发现"导致馆长被问责、博物馆赔钱。市面上的智能监控方案（海康等）要求更换摄像头，硬件成本上百万，博物馆无力承担。

## Demand Evidence

- 省级博物馆馆长亲口抱怨"出了事才发现"
- 已发生过藏品损坏、被盗、游客受伤等真实事故
- 馆长因此被问责并赔钱 — 这是职业风险级别的痛点
- 了解过海康等方案，因硬件成本上百万放弃
- 2人盯30路摄像头 = 形同虚设，结构性的监控盲区

## Status Quo

当前应对方式：2名安保人员轮班盯30路监控画面。每人同时看15路，实际上无法有效监控。出事后靠回放录像定位问题，属于事后追溯而非事前预防。曾考虑海康威视等传统智能监控方案，但需要更换全部摄像头，硬件成本上百万，超出预算放弃。

真正的竞争对手不是别的AI创业公司，是"2个人盯屏幕 + 出事后看回放"这个现状。

## Target User & Narrowest Wedge

**目标用户：** 省级博物馆馆长/安保负责人。关心藏品安全，因安全事故承担问责和经济损失。

**最小楔子：** 藏品防盗实时预警。有人在藏品区域出现异常行为（靠近、触碰、滞留、遮挡摄像头）时，秒级推送报警到手机。这是让馆长掏钱的钩子。

**完整产品：** 在防盗预警基础上，提供RAG智能问答、藏品盘点、事件趋势分析、仪表盘等增值功能，作为"买一送多"的差异化竞争力。

## Constraints

- 必须复用现有摄像头（RTSP拉流），零硬件改造成本
- 私有化部署，视频数据不出馆，满足文物安全合规要求
- 需要GPU服务器运行YOLO检测和大模型推理
- 博物馆IT能力有限，部署和运维必须极简（Docker一键启动）
- 定价远低于百万级，控制在博物馆容易审批的范围内

## Premises

1. ✅ 省级博物馆馆长是目标用户，"出了事才发现"导致问责赔钱，痛点真实
2. ✅ 传统方案要换摄像头，硬件上百万。纯软件方案复用现有摄像头是核心差异化
3. ✅ 产品包含全功能，但销售切入点聚焦"藏品防盗实时预警"，其他功能作为增值项
4. ✅ 定价远低于百万级，让博物馆不用走太复杂的审批流程

## Approaches Considered

### Approach A: 轻量SaaS（云端部署）
- 工作量: S
- 风险: 中
- 优点: 部署快、运维集中、可同时服务多馆
- 缺点: 视频上云合规风险、带宽成本高、博物馆不接受
- 复用: 现有后端代码

### Approach B: 私有化部署（All-in-One一体机）
- 工作量: M
- 风险: 低
- 优点: 数据不出馆、低延迟、合规无压力、部署极简
- 缺点: 每馆单独部署维护、需要GPU服务器
- 复用: 现有All-in-One Docker镜像、全部前后端代码

### Approach C: 混合架构（边缘+云端）
- 工作量: L
- 风险: 高
- 优点: 兼顾实时性和算力、长期最优
- 缺点: 架构复杂度翻倍、开发周期长、前期过度设计

## Recommended Approach

**Approach B: 私有化部署。**

理由：
1. 已有All-in-One Docker镜像，技术基础到位
2. 省级博物馆不会接受视频上云，私有化是行业默认期望
3. 风险最低，最快能签下第一个付费客户
4. 一台GPU服务器 + Docker部署 = 博物馆IT团队能接受的复杂度
5. 先用方案B跑通商业模式，有收入后再考虑方案C

## Open Questions

1. GPU服务器选型：博物馆机房放什么配置的服务器？NVIDIA T4够不够跑YOLO+vLLM？
2. 大模型选择：私有化部署用什么模型？Qwen-VL？InternVL？需要多大显存？
3. 报警通道：推送到手机用什么方式？微信？短信？钉钉？飞书？
4. 定价策略：软件授权费 + 年维护费？还是按摄像头路数收费？具体多少钱？
5. 第一个客户：那个馆长愿意做试点吗？能不能先免费部署换案例和口碑？
6. 竞品调研：除了海康，还有没有纯软件方案的竞品在做同样的事？

## Success Criteria

- 在一个省级博物馆完成部署，30路摄像头全部接入
- 藏品区域异常行为检测准确率 > 90%，误报率 < 5%
- 报警延迟 < 10秒（从异常发生到手机收到通知）
- 馆长认可效果并愿意付费
- 获得一个可公开的客户案例

## Distribution Plan

- 交付方式：GPU服务器预装系统 + Docker镜像，现场部署或远程指导
- 部署流程：docker compose up 一键启动，配置摄像头RTSP地址即可
- 更新方式：docker pull 新镜像 + restart
- CI/CD：GitHub Actions 自动构建镜像，推送到私有Registry

## Dependencies

- 博物馆提供摄像头RTSP接入权限和内网环境
- GPU服务器采购（可由博物馆采购或打包在方案中）
- 大模型选型和微调（藏品场景的异常行为识别）
- YOLO模型训练数据（博物馆场景的人体姿态、异常行为样本）

## The Assignment

**这周要做的一件事：** 联系那个馆长，问他一个问题："如果我能用你现有的摄像头，不换任何硬件，做到有人靠近藏品时10秒内手机收到报警，你愿意让我在你馆里免费试装一个月吗？"

如果他说愿意，你就有了第一个试点客户。如果他犹豫，问他在犹豫什么 — 那个犹豫就是你下一个要解决的问题。

不要继续写代码了。先拿到这个"是"。
